How it works

One script, one key, zero fumbling.

1

Write your blocks

A script is an ordered list of text blocks - one per beat of your demo. Add pacing with inline markers like [0.5] and [speed:40].

2

Focus any field

Your editor, a browser, a chat app - wherever the typing should land. TypeCue stays out of sight in the menu bar.

3

Press ⌃⌥X

The next block types itself, human-paced. Press again for the block after. Press mid-type to stop.

Built for people who type in front of people

Demo videos, tutorials, webinars, live presentations - whenever it has to come out right the first time.

Real keystrokes

Synthetic key events at the system level - indistinguishable from physical typing to the receiving app. No pasting, no injection tricks.

Direct the pace

Inline markers control the rhythm: [0.5] pauses, [speed:40] changes tempo mid-line, [enter] submits on purpose.

Natural rhythm

Subtle pace variation with pauses at spaces and punctuation, so the typing reads like a person, not a metronome.

Floating teleprompter

An always-on-top panel shows every block and where you are - typed, typing, next - without stealing focus from the app you're in.

Any language, any layout

Layout-aware typing with RTL support. Characters not on your keyboard layout arrive by direct Unicode entry - switching input sources mid-session just works.

Chat-safe line breaks

Multi-line blocks insert line breaks with Shift+Return by default, so nothing submits early in Claude, Cursor, or Slack.

One permission, explained

Accessibility access - the macOS API for sending keystrokes - and nothing else. No account, no network calls, no analytics. Read the source.

Plain-file scripts

Your scripts live in a readable JSON file on your Mac. Edit them by hand, sync them, or let your AI tools write them for you.

For AI agents and assistants

TypeCue is built to be driven by the tools you already use. Claude Code, Cursor, or any assistant with access to your machine can author scripts, pace them like a director, and control the app.

# Scripts (plain JSON - edit directly)
~/Library/Application Support/TypeCue/scripts.json
[{ "id", "name", "blocks": [{ "id", "text" }] }]

# Drive the app
open "typecue://activate-script?name=My%20demo"
open "typecue://reset-session"
open "typecue://reload"   # after editing scripts.json

# Observe it
~/Library/Application Support/TypeCue/state.json

The repo ships a ready-made agent skill: the schema, safety guardrails, and screencast-direction rules - typing speeds, pause lengths, where a beat belongs - so you can ask your assistant for things like:

"Turn this outline into a paced TypeCue script for my two-minute demo."

It writes the blocks, adds the markers, reloads the app. You press the key.

Questions

Why does TypeCue need Accessibility permission?

Accessibility is the macOS API that lets an app send keystrokes to other apps - it's the entire mechanism behind TypeCue, and the only permission it asks for. The onboarding includes a Test Pad so you can verify it actually works before your demo. TypeCue makes no network calls and collects nothing; the code is open if you want to check.

Does it work in password fields?

No - macOS blocks synthetic typing into secure fields system-wide, for every app. TypeCue detects this and flashes a warning instead of silently consuming your block. Type passwords manually; that's the OS working as intended.

Does it work in my language?

Yes. Typing is layout-aware (RTL included): characters on your active keyboard layout are sent as real keystrokes, and everything else - other alphabets, emoji - arrives by direct Unicode entry. Switching input sources mid-session is handled automatically.

Which apps does it type into?

Anything with a text field: browsers, editors like Cursor and VS Code, chat apps like Claude and Slack, terminals, TextEdit. If you can click into it and type, TypeCue can type there too.

How is this different from TextExpander or Keyboard Maestro?

Those expand snippets or run automations on demand. TypeCue plays an ordered script: one hotkey, the next block, in sequence, with human pacing you direct - built specifically for performing a demo, not for everyday shortcuts. It's also free and MIT licensed.

Does it pair with screen recorders?

Perfectly - TypeCue handles what happens during the take, recorders like Screen Studio handle capturing and polishing it. Scripted typing also means every retake is identical, which makes editing easier.

Is it really free?

Yes - free and open source under the MIT license. No paid tier, no account, no upsell. If it helps you ship a better demo, a star on GitHub is plenty.
